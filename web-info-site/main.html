1. O que são HTML, CSS e JavaScript?

HTML (HyperText Markup Language)

  • Função Principal: Estruturação do conteúdo web
  • Elementos Fundamentais:
    • Tags semânticas: <header>, <nav>, <main>, <article>, <section>, <footer>
    • Tags de conteúdo: <h1> a <h6>, <p>, <ul>, <ol>, <table>
    • Tags de mídia: <img>, <video>, <audio>
  • Versão Atual: HTML5 com suporte a recursos modernos como canvas, SVG e APIs web
<!DOCTYPE html> <html lang="pt-BR"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Exemplo de HTML Semântico</title> </head> <body> <!-- Estrutura semântica básica --> <header> <h1>Título Principal</h1> <nav> <ul> <li><a href="#">Início</a></li> <li><a href="#">Sobre</a></li> </ul> </nav> </header> <main> <article> <h2>Artigo Principal</h2> <section> <h3>Seção do Artigo</h3> <p>Conteúdo da seção...</p> <!-- Exemplo de mídia --> <img src="exemplo.jpg" alt="Descrição da imagem"> </section> </article> </main> <footer> <p>© 2025 Exemplo de Site</p> </footer> </body> </html>

CSS (Cascading Style Sheets)

  • Função Principal: Estilização e layout
  • Recursos Modernos:
    • Flexbox: Para layouts unidimensionais
    • Grid: Para layouts bidimensionais
    • Animações e Transições
    • Media Queries: Para design responsivo
  • Metodologias: BEM, SMACSS, OOCSS para organização do código
  • Pré-processadores: Sass, Less, Stylus para código mais manutenível

Como Trabalham Juntos

1. HTML fornece a estrutura base e o conteúdo

2. CSS transforma essa estrutura em um design atraente e responsivo

3. JavaScript adiciona comportamento dinâmico e interatividade

Este trio forma a base do desenvolvimento web moderno, permitindo criar desde simples páginas até aplicações web complexas.

2. Boas práticas profissionais

Acessibilidade Web

  • Princípios WCAG:
    • Perceptível: Informações visíveis para todos os sentidos
    • Operável: Interface funcional para diferentes formas de interação
    • Compreensível: Conteúdo claro e previsível
    • Robusto: Compatível com tecnologias assistivas
  • Implementações Práticas:
    • Textos alternativos para imagens
    • Contraste adequado de cores
    • Navegação por teclado
    • Legendas em vídeos

Código Limpo e Manutenção

  • Organização de Arquivos:
    • Estrutura de pastas clara (assets, components, styles)
    • Nomenclatura consistente
    • Separação de responsabilidades
  • Documentação:
    • Comentários explicativos
    • README detalhado
    • Documentação de API
    • Guia de estilo do código

Design Responsivo

  • Técnicas Principais:
    • Mobile-first design
    • Breakpoints estratégicos
    • Imagens flexíveis
    • Typography responsiva
  • Testes:
    • Múltiplos dispositivos
    • Diferentes navegadores
    • Velocidades de conexão variadas

Controle de Versão

  • Práticas com Git:
    • Commits semânticos
    • Branches organizados (main, develop, feature)
    • Code review
    • Resolução de conflitos
  • Fluxo de Trabalho:
    • GitFlow ou Trunk-Based Development
    • Pull requests
    • Integração contínua

Performance

  • Otimizações:
    • Minificação de código
    • Compressão de imagens
    • Lazy loading
    • Caching eficiente
  • Métricas:
    • Core Web Vitals
    • Tempo de carregamento
    • Performance Score

// Exemplo de API RESTful com Express const express = require('express'); const app = express(); app.use(express.json()); // Middleware de autenticação const autenticar = (req, res, next) => { const token = req.headers.authorization; if (!token) return res.status(401).json({ erro: 'Token não fornecido' }); // Verificação do token... next(); }; // Rotas da API app.get('/api/usuarios', autenticar, async (req, res) => { try { const usuarios = await Usuario.find(); res.json(usuarios); } catch (erro) { res.status(500).json({ erro: 'Erro ao buscar usuários' }); } }); app.post('/api/usuarios', autenticar, async (req, res) => { try { const novoUsuario = new Usuario(req.body); await novoUsuario.save(); res.status(201).json(novoUsuario); } catch (erro) { res.status(400).json({ erro: 'Erro ao criar usuário' }); } });

-- Exemplo de SQL Relacional -- Criação de tabelas CREATE TABLE usuarios ( id SERIAL PRIMARY KEY, nome VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE NOT NULL, criado_em TIMESTAMP DEFAULT CURRENT_TIMESTAMP ); CREATE TABLE posts ( id SERIAL PRIMARY KEY, usuario_id INTEGER REFERENCES usuarios(id), titulo VARCHAR(200) NOT NULL, conteudo TEXT, publicado BOOLEAN DEFAULT false ); -- Consultas comuns -- Buscar todos os posts de um usuário SELECT u.nome, p.titulo, p.conteudo FROM usuarios u JOIN posts p ON u.id = p.usuario_id WHERE u.id = 1; -- Inserir dados INSERT INTO usuarios (nome, email) VALUES ('João Silva', 'joao@email.com');
